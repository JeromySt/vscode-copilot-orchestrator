# OE0002 — Direct instantiation of DI-managed type outside Composition

**Severity:** Error  
**Category:** AiOrchestrator  

## Rule description

Types registered in the DI container must never be directly instantiated with `new` outside the
`AiOrchestrator.Composition` project. Directly constructing services breaks testability, bypasses
lifetime management, and hides dependencies.

## Non-compliant code

```csharp
// ❌ In AiOrchestrator.Core — creates a concrete service directly
public class Worker
{
    private readonly MyService _svc = new MyService(); // OE0002
}
```

## Compliant code

```csharp
// ✅ Inject via interface through the DI container
public class Worker
{
    private readonly IMyService _svc;
    public Worker(IMyService svc) => _svc = svc;
}

// ✅ Only AiOrchestrator.Composition may call new
namespace AiOrchestrator.Composition
{
    services.AddSingleton<IMyService, MyService>();
}
```

## Suppression

This rule has no safe suppression path. Refactor instead.
