# OE0020 — Public method has dynamic or object parameter type

**Severity:** Error  
**Category:** AiOrchestrator  

## Rule description

`dynamic` and `object` as parameter types defeat static typing, disable IDE tooling, and make
APIs hard to use correctly. Public methods must accept strongly-typed parameters or use generics.

## Non-compliant code

```csharp
// ❌ Dynamic / object parameter on public method
public class Serializer
{
    public string Serialize(dynamic value) { } // OE0020
    public string SerializeAny(object payload) { } // OE0020
}
```

## Compliant code

```csharp
// ✅ Strongly typed or generic parameter
public class Serializer
{
    public string Serialize<T>(T value, JsonSerializerContext ctx)
        => JsonSerializer.Serialize(value, typeof(T), ctx);
}
```

## Rationale

`object` parameters require runtime type checks and casts, which leads to brittle code and
unexpected `InvalidCastException` at runtime.
