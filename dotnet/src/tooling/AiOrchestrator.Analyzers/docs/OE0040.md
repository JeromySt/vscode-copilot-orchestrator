# OE0040 — Reflection-based JSON serialization

**Severity:** Error  
**Category:** AiOrchestrator  

## Rule description

`JsonSerializer.Serialize` and `JsonSerializer.Deserialize` overloads that do NOT accept a
`JsonSerializerContext` use runtime reflection to discover type metadata. This is:

- Incompatible with IL trimming (native AOT).
- Slower than source-generated serialization.
- Unable to work in fully trimmed deployments.

Always pass a `JsonSerializerContext` generated with `[JsonSerializable]`.

## Non-compliant code

```csharp
// ❌ Reflection-based — fails under trimming
var json = JsonSerializer.Serialize(myObject); // OE0040
var obj = JsonSerializer.Deserialize<MyType>(json); // OE0040
```

## Compliant code

```csharp
// ✅ Source-generated serialization context
[JsonSerializable(typeof(MyType))]
internal partial class MyJsonContext : JsonSerializerContext { }

var json = JsonSerializer.Serialize(myObject, MyJsonContext.Default.MyType);
var obj = JsonSerializer.Deserialize(json, MyJsonContext.Default.MyType);
```
