# OE0008 — async void method

**Severity:** Error  
**Category:** AiOrchestrator  

## Rule description

`async void` methods swallow exceptions — any exception thrown inside propagates to the
synchronization context and typically crashes the process. Use `async Task` instead so that
callers can observe exceptions.

The only exception to this rule is methods marked with `[AsyncEventHandler]` or those that match
the classic event-handler signature `(object sender, EventArgs e)`.

## Non-compliant code

```csharp
// ❌ async void swallows exceptions
public async void OnButtonClick() // OE0008
{
    await DoWorkAsync();
}
```

## Compliant code

```csharp
// ✅ Return Task so callers can observe exceptions
public async Task OnButtonClickAsync(CancellationToken cancellationToken = default)
{
    await DoWorkAsync(cancellationToken);
}

// ✅ Classic event handler — exempted
public async void Button_Click(object sender, EventArgs e)
{
    await DoWorkAsync();
}
```

## Code fix

The bundled fix changes the return type from `void` to `Task`.
