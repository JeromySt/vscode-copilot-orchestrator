# OE0046 — Logger message template uses string interpolation

**Severity:** Warning  
**Category:** AiOrchestrator  

## Rule description

Using C# string interpolation (`$"…{value}…"`) in a logger message template defeats structured
logging. The interpolated string is pre-rendered into a single opaque string; named properties
are lost and cannot be queried in log sinks (Seq, Application Insights, etc.).

Use named placeholder syntax instead (`"Processing {JobId}"`) and pass values as separate arguments.

## Non-compliant code

```csharp
// ❌ Interpolated string loses structured properties
_logger.LogInformation($"Processing job {jobId}"); // OE0046
```

## Compliant code

```csharp
// ✅ Named placeholder — queryable in structured log sinks
_logger.LogInformation("Processing job {JobId}", jobId);
```

## Suppression

If the logger method is known not to support structured logging, suppress at the call site:

```csharp
#pragma warning disable OE0046 // Legacy sink
_legacyLogger.Log($"Message: {value}");
#pragma warning restore OE0046
```
