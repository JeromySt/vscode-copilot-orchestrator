# Job 037 — Diagnose Bundle

**Tier:** T9
**Producer id:** `t9-diagnose`
**Dependencies:** `t2-redaction`, `t3-event-log`, `t4-audit`, `t7-plan-store`
**Estimated LOC:** ~900 production, ~1,300 tests

## Goal

Diagnose-bundle exporter per §3.18 + §3.31.1.3. Produces a portable `.aiodiag` archive containing redacted plan state, event log slices, audit log slices, host environment, and runtime telemetry. Implements DIAG-RECIP-* (pseudonymization integration with reversible mapping table).

## Detailed implementation requirements

### Public API surface

```csharp
namespace AiOrchestrator.Diagnose;

public sealed class Diagnoser
{
    public Diagnoser(IPlanStore planStore, IEventReader events, IAuditLog audit, IRedactor redactor, IPseudonymizer pseudonymizer, IFileSystem fs, IClock clock, IOptionsMonitor<DiagnoseOptions> opts);
    public ValueTask<AbsolutePath> ProduceBundleAsync(DiagnoseRequest request, CancellationToken ct);
}

public sealed record DiagnoseOptions
{
    public PseudonymizationMode PseudonymizationMode { get; init; } = PseudonymizationMode.Anonymous;
    public TimeSpan EventLogWindow { get; init; } = TimeSpan.FromHours(2);
    public bool IncludeAuditLog { get; init; } = true;
    public bool IncludeWorktreeListings { get; init; } = false;
    public bool IncludeProcessEnv { get; init; } = false;   // env may contain tokens; default off
}

public sealed record DiagnoseRequest { public required PlanId? PlanId { get; init; } public required AbsolutePath OutputPath { get; init; } public TimeSpan? EventLogWindow { get; init; } public string? Recipient { get; init; } /* for reversible mapping */ }
public enum PseudonymizationMode { Off, Anonymous, Reversible }

public interface IPseudonymizer
{
    ValueTask<string> PseudonymizeAsync(string original, PseudonymKind kind, CancellationToken ct);
    ValueTask<string?> ReverseAsync(string pseudonym, string recipientPubKeyFingerprint, byte[] privateKey, CancellationToken ct);
}
public enum PseudonymKind { UserName, Hostname, RepoUrl, FilePath, EmailAddress, IpAddress }

internal sealed class BundleManifest
{
    public required Version SchemaVersion { get; init; }   // "1"
    public required DateTimeOffset CreatedAt { get; init; }
    public required PseudonymizationMode PseudonymizationMode { get; init; }
    public required string? RecipientPubKeyFingerprint { get; init; }
    public required ImmutableDictionary<string, BundleEntry> Entries { get; init; }
    public required string DotnetRuntimeVersion { get; init; }
    public required string AioVersion { get; init; }
}
public sealed record BundleEntry { public required string Path { get; init; } public required long Bytes { get; init; } public required string Sha256 { get; init; } }
```

### Behavioral invariants

1. **INV-1** (DIAG-1) — Output is a `.zip` containing `manifest.json`, `plan.json` (redacted), `events.jsonl` (redacted), `audit.jsonl` (segments), `host.json` (OS, .NET version, AIO version), and (if reversible) `mapping.encrypted.bin`.
2. **INV-2** (DIAG-RECIP-1) — `PseudonymizationMode.Anonymous` replaces sensitive identifiers with stable pseudonyms (`user-A1B2`, `host-C3D4`, `repo-E5F6`); same input → same pseudonym within the bundle.
3. **INV-3** (DIAG-RECIP-2) — `PseudonymizationMode.Reversible` REQUIRES `Recipient` and a recipient public key in the configured trust store; mapping table is encrypted with RSA-OAEP-SHA-256 to recipient + AES-GCM session key (only recipient can reverse).
4. **INV-4** (DIAG-RECIP-3) — `PseudonymizationMode.Off` is FORBIDDEN unless `--allow-pii` flag was passed (warning recorded in manifest).
5. **INV-5** — All text content is run through `IRedactor` (job 006) FIRST, then through pseudonymizer.
6. **INV-6** — Bundle creation is auditable: `DiagnoseBundleProduced` event emitted with manifest hash + recipient fingerprint.
7. **INV-7** — Process env vars are excluded by default; `IncludeProcessEnv = true` requires explicit opt-in and emits a warning event.
8. **INV-8** — Bundle is deterministic: identical inputs produce byte-identical outputs (modulo timestamps, which are isolated to `manifest.json`).
9. **INV-9** — `Diagnoser` validates output path via `IPathValidator` (job 007).
