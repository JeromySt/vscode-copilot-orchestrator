# Job 041 — Acceptance Suite

**Tier:** T10
**Producer id:** `t10-acceptance`
**Dependencies:** ALL prior jobs (000-040)
**Estimated LOC:** ~2,000 tests (no new production code)

## Goal

Solution-wide acceptance gate: a single test project that asserts EVERY rule id from §§3.27–3.33 of the design doc has a corresponding `[ContractTest("<RULE-ID>")]` test somewhere in the solution. Acts as a regression gate — adding a new rule id to the design doc REQUIRES adding a contract test before merge. Also runs end-to-end scenarios that exercise multiple components.

## Source-doc sections

- §§3.27–3.33 ALL named rules
- §3.27 Composability acceptance scenarios

## Detailed implementation requirements

### Public API surface

```csharp
namespace AiOrchestrator.Acceptance;

public sealed class RuleIdCoverageGate
{
    public RuleIdCoverageGate(IFileSystem fs);
    public ValueTask<CoverageReport> RunAsync(AbsolutePath designDocPath, AbsolutePath solutionRoot, CancellationToken ct);
}

public sealed record CoverageReport
{
    public required ImmutableArray<string> AllRuleIds { get; init; }   // extracted from design doc
    public required ImmutableArray<string> CoveredRuleIds { get; init; }   // from [ContractTest("...")] attributes
    public required ImmutableArray<string> UncoveredRuleIds { get; init; }
    public required ImmutableArray<string> ExtraTestRuleIds { get; init; }   // tests for rules not in doc
    public bool Ok => UncoveredRuleIds.IsEmpty;
}

internal sealed class DesignDocRuleExtractor
{
    public ImmutableArray<string> Extract(string markdown);   // matches /\b(?:[A-Z]+-)+[A-Z0-9]+\b(?:-\d+)?/ within §3.27-3.33 sections
}

internal sealed class ContractTestScanner
{
    public ValueTask<ImmutableArray<string>> ScanAsync(AbsolutePath solutionRoot, CancellationToken ct);   // walks all .cs files for [ContractTest("...")] attribute literal
}
```

### Behavioral invariants

1. **INV-1** — `RuleIdCoverageGate.RunAsync` extracts rule ids from `docs/DOTNET_CORE_REARCHITECTURE_PLAN.md` §§3.27–3.33 sections.
2. **INV-2** — `ContractTestScanner` walks `tests/dotnet/**/*.cs` and collects every literal string passed to `[ContractTest(...)]`.
3. **INV-3** — `CoverageReport.Ok = true` iff `UncoveredRuleIds.IsEmpty`.
4. **INV-4** — `ExtraTestRuleIds` (tests for rules not in doc) is REPORTED but does NOT fail the gate (allows test-driven exploration).
5. **INV-5** — Gate test `RULE_COVERAGE_GATE_AllNamedRulesHaveTests` runs as a unit test in CI; failure blocks merge.
6. **INV-6** — End-to-end scenario tests exercise multi-component flows (e.g. "create plan → add jobs → finalize → run → verify SV → diagnose bundle" round-trip).
