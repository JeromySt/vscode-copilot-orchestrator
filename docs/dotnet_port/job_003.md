# Job 003 — Test Kit

**Tier:** T1 (contracts)
**Producer id:** `t1-testkit`
**Dependencies:** `t1-models`, `t1-abstractions`
**Estimated LOC:** ~900 production, ~600 tests

## Goal

Reusable test infrastructure consumed by every downstream test project. Includes deterministic in-memory implementations of every abstraction, property generators, the `[ContractTest]` attribute, the `CompositionFixture` collection, and contract-test base classes that ANY conforming concrete implementation must subclass to prove conformance to the abstraction's contract.

### Behavioral invariants

1. **INV-1** — Every interface in `AiOrchestrator.Abstractions` has a corresponding `*ContractTests` abstract base class in TestKit (reflection scan in `TKIT-1`).
2. **INV-2** — Every interface in `AiOrchestrator.Abstractions` has at least one in-memory implementation in TestKit (`TKIT-2`).
3. **INV-3** — `[ContractTest]` rule ids are globally unique across the touched test assemblies (`TKIT-3`).
4. **INV-4** — `InMemoryClock` advances ONLY when `Advance()` or `SetUtc()` is called; `IDelayProvider` consumers therefore never make real-time delays in tests.
5. **INV-5** — `RecordingEventBus` preserves publish order across concurrent publishers (uses a single `Channel`).
6. **INV-6** — `FakeProcessSpawner.Spawns` is a thread-safe append-only list.
7. **INV-7** — `CompositionFixture.Services` resolves every interface in `AiOrchestrator.Abstractions` to a non-null instance (registers TestKit fakes by default).
8. **INV-8** — Project is `<IsPackable>false</IsPackable>` and has `<IsTestProject>false</IsTestProject>` (it's a test-support library, not a test project).
