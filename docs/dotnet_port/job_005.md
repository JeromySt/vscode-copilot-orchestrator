# Job 005 — Configuration

**Tier:** T2
**Producer id:** `t2-configuration`
**Dependencies:** `t1-abstractions`, `t1-testkit`
**Estimated LOC:** ~450 production, ~550 tests

## Goal

Layered configuration provider implementing `IConfigurationProvider`. Layers (lowest precedence to highest): defaults → `appsettings.json` → `appsettings.<env>.json` → environment variables (prefix `AIO_`) → CLI flags → in-memory test override. Strongly-typed binding via Options source-generator. Validation via `IValidateOptions<T>`. Hot reload via change tokens.

### Behavioral invariants

1. **INV-1** — Layer precedence (low → high): defaults < appsettings.json < appsettings.{env}.json < env (`AIO_` prefix, `__` separator) < CLI < InMemory.
2. **INV-2** — Each options class has `[OptionsValidator]` and validates at startup (eager).
3. **INV-3** — `OnChange<T>` fires within 500 ms of an underlying source change; the handler receives the post-change value, never the pre-change value.
4. **INV-4** — Invalid value at bind time throws `OptionsValidationException` with the field path.
5. **INV-5** — Env-var translation: `AIO_Scheduler__Channel__Capacity=2048` ⇒ `SchedulerOptions.Channel.Capacity = 2048`.
6. **INV-6** — Defaults match those in §3.13.2.
7. **INV-7** — Hot-reload does NOT mutate already-bound `T` instances; downstream consumers must subscribe to `OnChange` to receive new values.
