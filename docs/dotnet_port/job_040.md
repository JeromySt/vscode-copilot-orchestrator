### Public API surface

```csharp
namespace AiOrchestrator.VsCode.Transport;

public sealed class VsCodeTransport : IAsyncDisposable
{
    public VsCodeTransport(NodeBindingsHost bindings, McpServer mcp, IClock clock, IOptionsMonitor<TransportOptions> opts, ILogger<VsCodeTransport> logger);
    public ValueTask<TransportSession> CreateSessionAsync(VsCodeWindowId windowId, CancellationToken ct);
}

public sealed record TransportOptions { public TimeSpan SessionIdleTimeout { get; init; } = TimeSpan.FromMinutes(30); public bool EnableProgressNotifications { get; init; } = true; }

public sealed class TransportSession : IAsyncDisposable
{
    public required VsCodeWindowId WindowId { get; init; }
    public required HandleScope Scope { get; init; }
    public ValueTask<JsonNode> InvokeToolAsync(string toolName, JsonElement parameters, CancellationToken ct);
    public IAsyncEnumerable<TransportEvent> WatchEventsAsync(CancellationToken ct);
    public ValueTask DisposeAsync();
}

public readonly record struct VsCodeWindowId(string Value);

public sealed record TransportEvent { public required string Kind { get; init; } public required JsonElement Payload { get; init; } public required DateTimeOffset At { get; init; } }
```

### Behavioral invariants

1. **INV-1** — `VsCodeTransport` is the ONLY .NET project with a reference to `Microsoft.VisualStudio.*`; analyzer enforces.
2. **INV-2** — Each VS Code window has its own `TransportSession` with a private `HandleScope` (job 36); session disposal disposes all scoped handles.
3. **INV-3** — Session idle timeout (default 30 min) auto-disposes session; tool invocation resets the idle timer.
4. **INV-4** — `WatchEventsAsync` multiplexes events from the orchestrator (plan progress, job state changes, log lines) into a single async iterator.
5. **INV-5** — Tool invocations route through `McpToolRegistry` (job 35); the wire format is JSON-RPC 2.0 framed.
6. **INV-6** — `EnableProgressNotifications = true` translates orchestrator events into VS Code progress notifications via the MCP `progress` method.
7. **INV-7** — `TransportSession` propagates `CancellationToken` from VS Code's cancellation source through to the orchestrator.
8. **INV-8** — Errors crossing the boundary preserve the .NET exception type name in `error.code`.
