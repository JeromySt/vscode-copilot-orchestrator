# Job 039 — Skew Manifest

**Tier:** T9
**Producer id:** `t9-skew-manifest`
**Dependencies:** `t2-clock`, `t4-audit`
**Estimated LOC:** ~1,300 production, ~1,800 tests
**Effort:** high (security-critical)

## Goal

Build-key skew manifest per §3.31.1.5 + §3.31.2.6. Tracks the active set of trusted audit-log signing keys via a periodically-rotated `release-manifest.signed.json` produced by an offline ceremony. Implements SKEW-MFST-* (manifest schema, time-bounded validity) and OFFLINE-ROOT-1..7 (M-of-N HSM 3-of-5, dual-root burn-in, 5-yr rotation, EmergencyRevocation, Sigstore-style transparency log).

## Source-doc sections

- §3.31.1.5 KEY-ROT-* + TRUST-ROOT-1..7 (governance)
- §3.31.2.6 Daemon update rollback (consumer of manifest)

### Behavioral invariants

1. **INV-1** (SKEW-MFST-1) — Manifest is fetched from `ManifestUrl` over HTTPS with certificate validation.
2. **INV-2** (SKEW-MFST-2) — Manifest is rejected if `SignedAt > Now + 5 min` (clock skew tolerance) OR `NotValidAfter < Now`.
3. **INV-3** (SKEW-MFST-3) — Manifest is rejected if `StaleAfter` elapsed since `SignedAt` and no newer manifest is available; emits `SkewManifestStale` warning.
4. **INV-4** (OFFLINE-ROOT-1) — At least `RequiredHsmSignatures` (default 3) of `KnownHsmPublicKeys` (default 5) must validly sign the manifest payload.
5. **INV-5** (OFFLINE-ROOT-2) — Burn-in: 5 HSM public keys are CONFIGURED at install time (not fetched from manifest); any new HSM keys require an emergency revocation + reissue path.
6. **INV-6** (OFFLINE-ROOT-3) — Manifest version is monotonically increasing; regression → `VersionRegression` rejection.
7. **INV-7** (OFFLINE-ROOT-4) — `EmergencyRevocation` is honored only if signed by ≥ `RequiredHsmSignatures` of `EmergencyRevocationPublicKeys` (a separate HSM set, intended to be air-gapped).
8. **INV-8** (OFFLINE-ROOT-5) — Manifest carries a `TransparencyLogProof`; if `TransparencyLogUrl` is configured, daemon verifies inclusion via Merkle proof.
9. **INV-9** (OFFLINE-ROOT-6) — Routine HSM key rotation occurs every 5 years (not enforced in code but documented as policy in `docs/SECURITY.md` § Key Ceremony).
10. **INV-10** (OFFLINE-ROOT-7) — Daemon (this code) NEVER signs anything with an HSM key; signing is the offline ceremony binary's job (`tools/key-ceremony`, job 043).
11. **INV-11** — Every fetched manifest is audited via `IAuditLog.AppendAsync` with verification result.
12. **INV-12** — `KeyCeremonyToolingStub` exists ONLY to keep the type referenced; analyzer fires if any non-tools/non-test assembly calls it.
