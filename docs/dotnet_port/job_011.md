### Behavioral invariants

1. **INV-1** — Each diagnostic is documented under `docs/OE####.md` (rationale, examples, suppression guidance).
2. **INV-2** — Each diagnostic ships ≥ 1 positive test (analyzer fires) and ≥ 1 negative test (analyzer silent on known-good fragment).
3. **INV-3** — Each diagnostic that has a deterministic fix ships a `CodeFixProvider` + ≥ 1 fix-application test.
4. **INV-4** — Analyzer pack is `IsPackable=true` and ships as a NuGet `analyzer/dotnet/cs/` payload.
5. **INV-5** — The analyzer pack is referenced by every `src/dotnet/AiOrchestrator.*` project (NOT test projects, NOT analyzer project itself) via `Directory.Build.targets`.
6. **INV-6** — Analyzer self-applies its own rules (eat-our-own-dogfood gate).
7. **INV-7** — DiagnosticIds are exposed as constants on `Diagnostics.cs` so other projects can reference them in suppressions if absolutely necessary.
