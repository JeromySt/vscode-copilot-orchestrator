# Job 021 — Concurrency: Per-User FIFO

**Tier:** T5
**Producer id:** `t5-conc-user`
**Dependencies:** `t2-clock`, `t3-event-bus`
**Estimated LOC:** ~400 production, ~700 tests

## Goal

Per-user FIFO concurrency limiter per §3.9.1. Bounded `SemaphoreSlim` per principal with FIFO admission ordering and `ConcurrencyUserQueued` / `ConcurrencyUserAdmitted` events for observability.

## Detailed implementation requirements

### Public API surface

```csharp
namespace AiOrchestrator.Concurrency.User;

public sealed class PerUserConcurrencyLimiter : IPerUserConcurrency, IAsyncDisposable
{
    public PerUserConcurrencyLimiter(IClock clock, IEventBus bus, IOptionsMonitor<UserConcurrencyOptions> opts);
    public ValueTask<UserAdmission> AcquireAsync(AuthContext principal, JobId job, CancellationToken ct);
    public ValueTask<int> GetActiveCountAsync(AuthContext principal, CancellationToken ct);
}

public sealed record UserConcurrencyOptions { [Range(1, 1000)] public int MaxConcurrentPerUser { get; init; } = 4; public int FifoQueueDepth { get; init; } = 1024; }

public sealed class UserAdmission : IAsyncDisposable
{
    public required AuthContext Principal { get; init; }
    public required JobId JobId { get; init; }
    public required DateTimeOffset AdmittedAt { get; init; }
    public ValueTask DisposeAsync();   // releases slot in FIFO order
}

public sealed class ConcurrencyUserQueued { public required AuthContext Principal { get; init; } public required JobId JobId { get; init; } public required int QueuePosition { get; init; } public required DateTimeOffset At { get; init; } }
public sealed class ConcurrencyUserAdmitted { public required AuthContext Principal { get; init; } public required JobId JobId { get; init; } public required TimeSpan WaitTime { get; init; } public required DateTimeOffset At { get; init; } }
public sealed class UserConcurrencyQueueFullException : Exception { public required AuthContext Principal { get; init; } public required int QueueDepth { get; init; } }
```

### Behavioral invariants

1. **INV-1** — Each unique `AuthContext.UserId` has its own bounded `SemaphoreSlim(MaxConcurrentPerUser, MaxConcurrentPerUser)` and a FIFO ordered queue.
2. **INV-2** — Admission is strictly FIFO per user: the longest-waiting `AcquireAsync` call wins.
3. **INV-3** — `ConcurrencyUserQueued` event published exactly once per AcquireAsync that does NOT admit immediately; `ConcurrencyUserAdmitted` published exactly once per admission with `WaitTime`.
4. **INV-4** — Queue depth bounded at `Options.FifoQueueDepth`; overflow raises `UserConcurrencyQueueFullException` (caller decides to retry).
5. **INV-5** — `UserAdmission.DisposeAsync` releases exactly one slot for that user.
6. **INV-6** — Cancellation of `AcquireAsync` removes the waiter from the queue without affecting FIFO order of others.
