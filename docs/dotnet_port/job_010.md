## Goal

`IProcessSpawner` + `IProcessHandle` implementing argv-vector spawn (no shell), per-process resource limits (Linux cgroups v2 + setrlimit; Windows job objects), PTY allocation (libpty / ConPTY), unified stdout/stderr async via `System.IO.Pipelines`, cancellation → SIGTERM → SIGKILL with grace, env scrubbing via `EnvScope`, crash-dump capture on abnormal exit.

### Behavioral invariants

1. **INV-1** — `SpawnAsync` always passes `argv` as a vector; never invokes a shell. Verified by `PROC-1` (Roslyn: no `UseShellExecute = true`).
2. **INV-2** — Cancellation flow: on cancel, send SIGTERM (Unix) / `GenerateConsoleCtrlEvent(CTRL_BREAK_EVENT)` (Windows); after grace (default 5 s, configurable), SIGKILL/`TerminateProcess`. Verified by `PROC-2`.
3. **INV-3** — `EnvScope.InheritOther = false` ⇒ child receives ONLY keys in `Allowed` (no `PATH`, no `HOME` unless explicit).
4. **INV-4** — Stdout and stderr are exposed as separate `PipeReader`s; both are flushed on process exit.
5. **INV-5** — `WaitForExitAsync` is observable from multiple awaiters (multi-shot).
6. **INV-6** — On abnormal exit (signal/segfault), `IProcessLifecycle.CaptureCrashDumpAsync` is invoked; failure to capture is logged but not thrown.
7. **INV-7** — PTY allocation: `IPtyAllocator` returns master+slave fds/handles; the spawned child gets slave attached as stdin/stdout/stderr.
8. **INV-8** — `DisposeAsync` is idempotent and force-kills if process is still alive.
9. **INV-9** — Resource limits are applied BEFORE `execve` on Linux (via fork-then-setrlimit-then-exec), not after.
