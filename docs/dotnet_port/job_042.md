# Job 042 — Benchmarks

**Tier:** T10
**Producer id:** `t10-benchmarks`
**Dependencies:** ALL prior jobs
**Estimated LOC:** ~1,200 tests/benchmarks

## Goal

BenchmarkDotNet performance suite per §3.31.3.2 (SLO-ENV-*). Measures hot paths against documented SLOs in a controlled environment. Implements SLO-ENV-* (precise environment specification: machine class, NUMA topology, .NET version, RyuJIT settings, GC mode, frequency scaling off). Produces a CI-tracked baseline + regression alarm.

```csharp
internal sealed class SloEnvConfig : ManualConfig
{
    public SloEnvConfig();   // Adds Job_Net10_RyuJIT_ServerGC, EnvironmentValidator (asserts NUMA pinned, freq scaling off if SLO-ENV strict mode), MarkdownExporter, JsonExporter, RankColumn, MemoryDiagnoser
}

public sealed record SloTarget { public required string BenchmarkId { get; init; } public required TimeSpan P50 { get; init; } public required TimeSpan P99 { get; init; } public required long? MaxAllocatedBytes { get; init; } }
internal sealed class SloRegressionGate
{
    public ValueTask<RegressionReport> RunAsync(AbsolutePath benchmarkResultsDir, AbsolutePath baselineFile, CancellationToken ct);
}
public sealed record RegressionReport { public required ImmutableArray<RegressionFinding> Regressions { get; init; } public bool Ok => Regressions.IsEmpty; }
public sealed record RegressionFinding { public required string BenchmarkId { get; init; } public required TimeSpan BaselineP99 { get; init; } public required TimeSpan CurrentP99 { get; init; } public required double DeltaPercent { get; init; } }
```

### Behavioral invariants

1. **INV-1** (SLO-ENV-1) — `SloEnvConfig` documents and (where possible) ASSERTS: .NET 10 RyuJIT, ServerGC, no tiered compilation timing skew, fixed frequency, NUMA-pinned worker.
2. **INV-2** (SLO-ENV-2) — Benchmark runs SHALL fail in CI if `EnvironmentValidator` reports environmental skew (e.g. throttled CPU); a soft mode allows local dev runs.
3. **INV-3** (SLO-ENV-3) — Each benchmark records P50 + P99 + allocated bytes; results emitted as JSON for CI ingestion.
4. **INV-4** — `SloRegressionGate` compares new run against `tests/dotnet/AiOrchestrator.Benchmarks/baselines/<benchmarkId>.json`; >10% P99 regression fails the gate.
5. **INV-5** — Allocation regression: benchmarks tagged `[MemoryDiagnoser]`; >5% allocation regression fails the gate.
6. **INV-6** — Benchmark suite is excluded from the standard `dotnet test` run (uses a separate `bench` target).
