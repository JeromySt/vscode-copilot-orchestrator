# Job 007 — Path Validator

**Tier:** T2
**Producer id:** `t2-path-validator`
**Dependencies:** `t1-abstractions`, `t1-testkit`
**Estimated LOC:** ~650 production (incl. native interop), ~700 tests

## Goal

Implement `AiOrchestrator.Paths` — `IPathValidator` enforcing **PATH-VAL-*** (§3.6). On Linux/macOS uses true `openat(2)` semantics via `[LibraryImport]`; on Windows uses reparse-point-aware enumeration. Defends against TOCTOU symlink-replace, ADS-suffix bypass on Windows, and `..` traversal.

### Behavioral invariants

1. **INV-1** — `AssertSafe` canonicalizes via `Path.GetFullPath` then verifies `result.StartsWith(allowedRoot.Value, StringComparison.Ordinal)` (case-sensitive on Linux/macOS, case-insensitive on Windows).
2. **INV-2** — On Linux, `OpenReadUnderRootAsync` opens parent directory with `O_DIRECTORY|O_NOFOLLOW`, then `openat(parentFd, name, O_NOFOLLOW)`; closes parent fd in a `try/finally`.
3. **INV-3** — On Windows, `OpenReadUnderRootAsync` rejects paths whose final segment contains `:` (ADS), `*`, `?`, or `<>"|`.
4. **INV-4** — Any path containing `..` after canonicalization throws `PathTraversalException`.
5. **INV-5** — `RepoRelativePath` validator confirms a path resolves within `repoRoot` and never traverses out via `..` or via symlink (`AssertSafe` re-checks under symlink-aware `openat`).
6. **INV-6** — `PathTraversalException` carries the offending segment and the attempted+allowed paths for diagnostics.
7. **INV-7** — Native interop via `[LibraryImport]` (source-generated P/Invoke); zero `[DllImport]`.
8. **INV-8** — Zero allocations on the success path (`ReadOnlySpan<char>`).
