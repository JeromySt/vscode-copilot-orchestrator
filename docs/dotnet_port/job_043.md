# Job 043 — Release Pipeline

**Tier:** T10
**Producer id:** `t10-release-pipeline`
**Dependencies:** ALL prior jobs (000-042)
**Estimated LOC:** ~1,500 (tooling + workflows)
**Effort:** high (security-critical)

## Goal

Release pipeline + key-ceremony tooling per §3.31.1.5 + §3.31.2.6. Two deliverables:

1. **`tools/key-ceremony/`** — offline binary that runs on an air-gapped machine, accepts the proposed `release-manifest.unsigned.json`, prompts the M-of-N HSM operators to sign, produces `release-manifest.signed.json`, and (optionally) submits to a Sigstore-style transparency log.
2. **`.github/workflows/`** — CI/CD pipelines: PR (build, test, coverage, analyzer, format), main (build, test, NuGet pack, sign with timestamp), release (consume signed manifest from offline ceremony, publish to GitHub Releases + NuGet + VS Code Marketplace).

### Public API surface

```csharp
// tools/key-ceremony/KeyCeremony.csproj
namespace AiOrchestrator.Tools.KeyCeremony;

public sealed class KeyCeremonyProgram
{
    public static Task<int> Main(string[] args);
}

internal sealed class CeremonyOrchestrator
{
    public CeremonyOrchestrator(IHsmClient hsm, IFileSystem fs, IClock clock, ILogger<CeremonyOrchestrator> logger);
    public ValueTask<CeremonyResult> RunAsync(CeremonyRequest request, CancellationToken ct);
}

public sealed record CeremonyRequest
{
    public required AbsolutePath UnsignedManifestPath { get; init; }
    public required AbsolutePath OutputSignedPath { get; init; }
    public required ImmutableArray<HsmOperatorId> RequiredSigners { get; init; }   // 3 of 5 must sign
    public required string CeremonyTranscriptPath { get; init; }
    public bool SubmitToTransparencyLog { get; init; } = true;
    public string? TransparencyLogUrl { get; init; }
}
public sealed record CeremonyResult { public required AbsolutePath SignedManifestPath { get; init; } public required ImmutableArray<HsmOperatorId> ActualSigners { get; init; } public required string? TransparencyLogReceipt { get; init; } public required AbsolutePath TranscriptPath { get; init; } }

public interface IHsmClient
{
    ValueTask<HsmDeviceInfo> ConnectAsync(HsmOperatorId operator_, CancellationToken ct);
    ValueTask<byte[]> SignAsync(HsmOperatorId operator_, byte[] payloadHash, CancellationToken ct);
    ValueTask DisconnectAsync(HsmOperatorId operator_, CancellationToken ct);
}
public sealed record HsmDeviceInfo { public required string DeviceSerial { get; init; } public required string FirmwareVersion { get; init; } public required byte[] PublicKey { get; init; } public required string KeyId { get; init; } }
public readonly record struct HsmOperatorId(string Value);
```

### Behavioral invariants

1. **INV-1** (CER-1) — `KeyCeremonyProgram` runs ONLY on an air-gapped host (refuses to start if any network interface other than `lo` is up; can be overridden with `--allow-network` for staging).
2. **INV-2** (CER-2) — Each HSM signature is requested individually with operator confirmation; no batch-signing.
3. **INV-3** (CER-3) — Ceremony produces a transcript file with timestamps, operator ids, device serials, hash of payload signed; transcript is written to a separate filesystem path so failures don't lose audit.
4. **INV-4** (CER-4) — Output `release-manifest.signed.json` schema matches `SignedReleaseManifest` (job 39); validation is run before write.
5. **INV-5** (CER-5) — Optional transparency-log submission produces a Merkle inclusion receipt, embedded into the manifest as `TransparencyLogProof`.
6. **INV-6** (CI-PR) — `.github/workflows/pr.yml` runs: `dotnet restore`, `dotnet build -warnaserror`, `dotnet test --collect:"XPlat Code Coverage"`, `pwsh ./scripts/dotnet/check-coverage.ps1`, `pwsh ./scripts/dotnet/check-analyzers.ps1`, `pwsh ./scripts/dotnet/check-banned-apis.ps1`, `pwsh ./scripts/dotnet/check-public-api.ps1`, `pwsh ./scripts/dotnet/check-composition.ps1` for every project, the rule-id coverage gate (job 41), CodeQL.
7. **INV-7** (CI-MAIN) — `.github/workflows/main.yml` additionally NuGet-packs each `AiOrchestrator.*` project, signs the packages with the build cert (timestamp via RFC 3161), uploads to staging feed.
8. **INV-8** (CI-RELEASE) — `.github/workflows/release.yml` is MANUAL-trigger only; takes `release-manifest.signed.json` URL/path as input; verifies signatures (job 39); promotes staged packages to `nuget.org`; uploads daemon binaries to GitHub Releases.
9. **INV-9** (CI-RELEASE-VS) — Separate VS Code marketplace publish step uses `vsce` with a marketplace PAT stored in repo secrets.
10. **INV-10** — All CI workflows pin action versions by SHA (no floating tags); enforced by Dependabot.
11. **INV-11** — Release workflow REFUSES to publish if the SkewManifest from job 39 doesn't observe the new release manifest's TrustedAuditPubKeys.
