# Job 013 — Event Log (T1/T2/T3 Persistent Store)

**Tier:** T3
**Producer id:** `t3-event-log`
**Dependencies:** `t2-filesystem`, `t2-clock`, `t2-redaction`, `t3-event-bus`
**Estimated LOC:** ~1,400 production, ~1,800 tests

## Goal

Three-tier persistent event log per §3.4.4 + v1.2/v1.3 hardenings. T1 hot ring buffer (shared memory), T2 append-only file (CRC32C-per-record + monotonic `recordSeq`), T3 cold compressed archive. Implements split-record handling (T2-READ-1..11), bounded reassembly (T2-READ-11), monotonic `recordSeq` gap detection, SUB-3 replay-then-live seam, per-plan disk cap (DISK-PLAN-*).

### Behavioral invariants

1. **INV-1** (T2-LOG-1) — Each frame: `[u32 length][u64 recordSeq][payload bytes][u32 crc32c-of-(length+recordSeq+payload)]`.
2. **INV-2** (T2-LOG-2) — `recordSeq` is strictly monotonically increasing; `RecordSeqRegression` is fatal.
3. **INV-3** (T2-READ-1) — Reader resumes from the last record whose CRC matched; partially-written tail is skipped.
4. **INV-4** (T2-READ-11) — Bounded reassembly buffer (default 16 MiB / 5 s); overflow emits `EventLogReassemblyAbandoned { recordSeq, bytesBuffered, elapsedMs }`.
5. **INV-5** (T2-READ-11) — `recordSeq` gap of > 1 emits `EventLogRecordSeqGap { lastSeen, nextSeen }` and returns to caller as a sentinel.
6. **INV-6** (SUB-3) — `ReadReplayAndLiveAsync` produces a single contiguous stream with NO gap and NO duplicate at the replay→live boundary, validated by `SUB_3_NoGapNoDup`.
7. **INV-7** (DISK-PLAN-1) — Per-plan reservation: if `TryReserve` fails, the publish fails fast with `DiskQuotaExceeded { planId, requested, current, cap }`.
8. **INV-8** — All file I/O via `IFileSystem`; CRC32C via `System.IO.Hashing.Crc32C`.
9. **INV-9** — Reader is allocation-free per record on the happy path (`ReadOnlySequence<byte>`).
