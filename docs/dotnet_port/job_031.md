### Behavioral invariants

1. **INV-1** — Phase order is fixed: Setup → Prechecks → Work → Postchecks → Commit → ForwardIntegration → Done.
2. **INV-2** — Failure in any phase records a `JobAttemptRecorded` mutation with `EndedAtPhase` and `FailureReason`.
3. **INV-3** (HEAL-RESUME-1) — `HealOrResumeStrategy.Decide`:
    - **TransientNetwork / TransientFileLock**: `PhaseResume` from the same phase (no AI invocation, just retry).
    - **AgentMaxTurnsExceeded / AgentNonZeroExit / AnalyzerOrTestFailure**: `AutoHeal` (re-run Work with healing instructions) IFF `AutoHeal=true` AND `prevAttempts < MaxAutoHealAttempts`.
    - **MergeConflict**: `AutoHeal` IFF `AutoHeal=true`; otherwise `GiveUp`.
    - **RemoteRejected**: `GiveUp`.
    - **Internal**: `GiveUp`.
4. **INV-4** (HEAL-RESUME-2) — Auto-heal NEVER repeats a successful prior phase (e.g. successful Prechecks are not re-run when only Postchecks failed).
5. **INV-5** (HEAL-RESUME-3) — `MaxAutoHealAttempts` (default 3) caps the heal retries; subsequent failures are `GiveUp`.
6. **INV-6** (DISK-PLAN-1 integration) — Commit phase reserves disk via `PerPlanDiskCap.TryReserve(planId, estimatedBytes)` BEFORE writing; failure raises `DiskQuotaExceededException` and aborts commit phase with `Failed`.
7. **INV-7** — Setup phase allocates worktree lease (job 19) for the duration of all subsequent phases until ForwardIntegration completes.
8. **INV-8** — All phases run within the allocated worktree; writes go through `CasLeaseStore.EnforceWriteWithTokenAsync` to enforce LS-INF-* invariants.
9. **INV-9** — Commit phase invokes `IHookGateClient.CheckInAsync` BEFORE `git commit` for any pre-commit hook.
10. **INV-10** — `expectsNoChanges = true` means commit succeeds iff there is NO diff; otherwise fails (per §3.12.6).
11. **INV-11** — All phase timeouts honored; on timeout the phase fails with `PhaseFailureKind.Timeout`.
12. **INV-12** — `CancellationToken` propagation: cancellation aborts the current phase within ≤ 1 s.
