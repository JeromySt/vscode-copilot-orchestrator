# Job 004 — Logging & Telemetry

**Tier:** T2 (cross-cutting)
**Producer id:** `t2-logging`
**Dependencies:** `t1-abstractions`, `t1-testkit`
**Estimated LOC:** ~500 production, ~600 tests

## Goal

Implement `AiOrchestrator.Logging`: structured logger on `Microsoft.Extensions.Logging.Abstractions` with custom JSON-line formatter, ambient correlation context (AsyncLocal), and an OpenTelemetry-backed `ITelemetrySink`. Redaction is applied at the formatter (job 06 supplies `IRedactor`).

public sealed class CompactJsonFormatter : ConsoleFormatter

### Behavioral invariants

1. **INV-1** — Every emitted log line is a single line of valid UTF-8 JSON terminated by `\n`.
2. **INV-2** — Every log line contains `timestamp`, `level`, `category`, `message`, and at minimum the ambient keys `traceId`, `spanId` when an Activity is in scope.
3. **INV-3** — Every string-valued property is passed through `IRedactor.Redact` before write.
4. **INV-4** — `AmbientContext` uses `AsyncLocal<ImmutableDictionary<string, object>>`; pushing returns a disposer that restores the prior snapshot atomically.
5. **INV-5** — `AmbientContext` does NOT leak across xUnit test cases (verified by `LOG-7`).
6. **INV-6** — `OtlpTelemetrySink.Enabled = false` ⇒ all `Record*` calls are zero-allocation no-ops.
7. **INV-7** — Tag dictionaries are copied defensively before being handed to OTel.
8. **INV-8** — Logger does NOT swallow formatter exceptions; on failure it writes a single fallback `LoggerFormatterFailure` line and rethrows.
