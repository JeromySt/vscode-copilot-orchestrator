### Behavioral invariants

1. **INV-1** — Every verb has both `--json` (machine-readable) and default human output.
2. **INV-2** — Exit codes follow `CliExitCodes` constants and are documented in `--help`.
3. **INV-3** — `--help` output for every verb passes a snapshot test (no accidental UX regressions).
4. **INV-4** — JSON output uses source-generated `JsonTypeInfo<T>` (no reflection serializer).
5. **INV-5** — Color output respects `NO_COLOR` env var and `--no-color` flag.
6. **INV-6** — `plan run` blocks until plan terminates; exit code reflects final plan status.
7. **INV-7** — `plan diagnose` produces a portable bundle (job 37) with redactor + pseudonymizer applied.
8. **INV-8** — All file paths from CLI arguments are validated via `IPathValidator` (job 007).
