# Job 028 — Plan Store

**Tier:** T7
**Producer id:** `t7-plan-store`
**Dependencies:** `t2-filesystem`, `t2-clock`, `t3-event-bus`, `t7-plan-models`
**Estimated LOC:** ~1,200 production, ~1,600 tests

## Goal

`IPlanStore` — durable plan storage per §3.12 + §3.31.2.2. Persists plan state as a journaled append-only log + checkpoint snapshots; supports replay-and-live reads (SUB-3 reuse), atomic mutation via journal entries, and content-hash idempotency (RW-2-IDEM) for retried writes.

```csharp
namespace AiOrchestrator.Plan.Store;

public sealed class PlanStore : IPlanStore, IAsyncDisposable
{
    public PlanStore(AbsolutePath storeRoot, IFileSystem fs, IClock clock, IEventBus bus, IOptionsMonitor<PlanStoreOptions> opts, ILogger<PlanStore> logger);
    public ValueTask<PlanId> CreateAsync(Plan initialPlan, IdempotencyKey idemKey, CancellationToken ct);
    public ValueTask<Plan?> LoadAsync(PlanId id, CancellationToken ct);
    public ValueTask MutateAsync(PlanId id, PlanMutation mutation, IdempotencyKey idemKey, CancellationToken ct);
    public ValueTask CheckpointAsync(PlanId id, CancellationToken ct);
    public IAsyncEnumerable<Plan> ListAsync(CancellationToken ct);
    public IAsyncEnumerable<PlanMutation> ReadJournalAsync(PlanId id, long fromSeq, CancellationToken ct);
    public IAsyncEnumerable<Plan> WatchAsync(PlanId id, CancellationToken ct);   // SUB-3: replay snapshot then live mutations
}

public sealed record PlanStoreOptions { public int CheckpointAfterMutations { get; init; } = 100; public TimeSpan CheckpointAfterTime { get; init; } = TimeSpan.FromMinutes(5); }

public abstract record PlanMutation { public required long Seq { get; init; } public required IdempotencyKey IdemKey { get; init; } public required DateTimeOffset At { get; init; } }
public sealed record JobAdded(long Seq, IdempotencyKey IdemKey, DateTimeOffset At, JobNode Node) : PlanMutation { }
public sealed record JobRemoved(long Seq, IdempotencyKey IdemKey, DateTimeOffset At, JobId JobId) : PlanMutation { }
public sealed record JobDepsUpdated(long Seq, IdempotencyKey IdemKey, DateTimeOffset At, JobId JobId, ImmutableArray<JobId> NewDeps) : PlanMutation { }
public sealed record JobStatusUpdated(long Seq, IdempotencyKey IdemKey, DateTimeOffset At, JobId JobId, JobStatus NewStatus, JobPhase NewPhase) : PlanMutation { }
public sealed record JobAttemptRecorded(long Seq, IdempotencyKey IdemKey, DateTimeOffset At, JobId JobId, JobAttempt Attempt) : PlanMutation { }
public sealed record PlanStatusUpdated(long Seq, IdempotencyKey IdemKey, DateTimeOffset At, PlanStatus NewStatus) : PlanMutation { }

public readonly record struct IdempotencyKey(string Value)
{
    public static IdempotencyKey FromContent(ReadOnlySpan<byte> content);   // SHA-256 of canonical JSON
    public static IdempotencyKey FromGuid(Guid value);
}

internal sealed class PlanJournal : IAsyncDisposable
{
    public PlanJournal(AbsolutePath path, IFileSystem fs, IClock clock);
    public ValueTask AppendAsync(PlanMutation mutation, CancellationToken ct);
    public IAsyncEnumerable<PlanMutation> ReadFromAsync(long fromSeq, CancellationToken ct);
    public ValueTask<bool> ContainsIdempotencyKeyAsync(IdempotencyKey key, CancellationToken ct);   // RW-2-IDEM lookup
}

internal sealed class PlanCheckpointer
{
    public PlanCheckpointer(AbsolutePath path, IFileSystem fs);
    public ValueTask WriteAsync(Plan plan, long upToSeq, CancellationToken ct);   // atomic temp+rename
    public ValueTask<(Plan plan, long upToSeq)?> LoadLatestAsync(CancellationToken ct);
}

public sealed class IdempotencyConflictException : Exception { public required IdempotencyKey Key { get; init; } public required PlanMutation StoredMutation { get; init; } public required PlanMutation NewMutation { get; init; } }
```

### Behavioral invariants

1. **INV-1** (PLAN-STORE-1) — Plan state is reconstructed by loading the latest checkpoint and replaying journal entries with `Seq > checkpointUpToSeq`.
2. **INV-2** (RW-2-IDEM-1) — Every mutation carries an `IdempotencyKey`. If the key already exists in the journal AND the stored mutation has the same content hash, the mutation is a no-op (idempotent retry succeeds).
3. **INV-3** (RW-2-IDEM-2) — If the key exists with DIFFERENT content, raises `IdempotencyConflictException` with both stored and new mutation for diagnosis.
4. **INV-4** (RW-2-IDEM-3) — `IdempotencyKey.FromContent` computes SHA-256 over canonical JSON of mutation payload; deterministic across processes.
5. **INV-5** — `MutateAsync` is the only path to change plan state; calling code never serializes a `Plan` directly.
6. **INV-6** — Checkpoint is atomic (write `<id>.checkpoint.tmp`, fsync, rename); journal is fsync-on-append.
7. **INV-7** — `WatchAsync` reuses SUB-3 from job 13 — snapshot at subscription time then replays journal entries from the snapshot's seq onward, then transitions to live with no gap and no dup.
8. **INV-8** — Journal entries reproduce in `Seq` order; gap or out-of-order detected at load time and raises `PlanJournalCorruptedException`.
9. **INV-9** — Auto-checkpoint after `CheckpointAfterMutations` mutations OR `CheckpointAfterTime` since last checkpoint.
10. **INV-10** — `ListAsync` enumerates plans by scanning the store root; never holds all plans in memory.
