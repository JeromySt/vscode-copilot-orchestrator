## Detailed implementation requirements

### Public API surface

```csharp
namespace AiOrchestrator.Eventing;

public sealed class EventBus : IEventBus, IAsyncDisposable
{
    public EventBus(IClock clock, IRedactor redactor, ILogger<EventBus> logger, IOptionsMonitor<EventBusOptions> opts);
    public ValueTask PublishAsync<TEvent>(TEvent @event, CancellationToken ct) where TEvent : notnull;
    public IAsyncDisposable Subscribe<TEvent>(EventFilter filter, Func<TEvent, CancellationToken, ValueTask> handler) where TEvent : notnull;
    public ValueTask DisposeAsync();
}

public sealed record EventBusOptions { [Range(1, 1_000_000)] public int PerSubscriptionBufferSize { get; init; } = 1024; public BackpressureMode Backpressure { get; init; } = BackpressureMode.Wait; public bool EnableDedup { get; init; } = false; public TimeSpan DedupWindow { get; init; } = TimeSpan.FromSeconds(5); }
public enum BackpressureMode { Wait, DropOldest, DropNewest, Throw }

internal sealed class Subscription<TEvent> : IAsyncDisposable
{
    public Subscription(AuthContext principal, EventFilter filter, Func<TEvent, CancellationToken, ValueTask> handler, Channel<TEvent> channel);
    public AuthContext PrincipalAtSubscribe { get; }   // EVT-AUTH-1: pinned at subscribe time
}

internal sealed class AuthContextFilter
{
    public bool Matches(EventFilter filter, EventEnvelope envelope, AuthContext currentPrincipal);
    // EVT-AUTH-2: filter evaluation uses filter.SubscribingPrincipal (pinned), NOT currentPrincipal
}
```

### Behavioral invariants

1. **INV-1** (EVT-AUTH-1) — A subscription pins the `AuthContext` of the principal at the time of `Subscribe()`; later context changes do NOT widen the filter.
2. **INV-2** (EVT-AUTH-2) — Even if a different principal calls into the bus later, events delivered to a subscription are filtered against the pinned principal's scopes.
3. **INV-3** — `PublishAsync` applies `IRedactor.Redact` to all string-valued fields of the envelope payload before fan-out.
4. **INV-4** (CONC-CHAN-2) — When `EnableDedup = true`, events with the same `(EventType, DedupKey)` published within `DedupWindow` deliver only once.
5. **INV-5** — Backpressure: per-subscription channel bounded; on full, applies `BackpressureMode` policy and emits `EventBusSubscriptionLagged` event with `(subscriptionId, droppedCount, mode)`.
6. **INV-6** — `Subscribe` returns an `IAsyncDisposable`; disposing drains the channel, removes the subscription, and never blocks the publisher thread.
7. **INV-7** — Bus uses `Channel<T>` exclusively for cross-thread; zero locks on the publish path.
8. **INV-8** — `DisposeAsync` waits up to 5 s for all subscriptions to drain, then forces close.
