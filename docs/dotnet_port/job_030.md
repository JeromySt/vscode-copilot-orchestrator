## Goal

Plan scheduler per §3.12.5 + §3.31.2.4 + §3.31.4.2. Drives DAG execution: walks ready set, admits via concurrency limiters, emits scheduling events, applies channel-bounds rules (CONC-CHAN-* with dedup-by-event-key), and resolves T22/T14 race rules deterministically (snapshot-validation injection vs reshape).

### Behavioral invariants

1. **INV-1** (SCHED-RDY-1) — `ReadySet.ComputeReady` returns the set of pending jobs whose every predecessor is in `Succeeded` state.
2. **INV-2** (SCHED-RDY-2) — Job becomes `Blocked` if any predecessor is in a terminal-failure state (`Failed` or `Canceled`); blocked jobs do NOT become Ready ever again.
3. **INV-3** (CONC-CHAN-1) — Both `ReadyChannel` and `ScheduledChannel` are bounded (default 1024 / 256); writers block on full.
4. **INV-4** (CONC-CHAN-2) — When `EnableEventDedup = true`, schedule events with the same `(planId, jobId, eventType)` within `DedupWindow` (default 5 s) deliver only once.
5. **INV-5** (T22-RULE) — Snapshot validation node injection happens AFTER all leaf jobs become Ready and BEFORE any leaf job's commit phase completes; if a leaf finishes commit FIRST, the SV injection still observes the correct dependency snapshot.
6. **INV-6** (T14-RULE) — A reshape that adds a new leaf during plan execution forces re-injection of SV node dependencies; old SV depends-on edges are atomically replaced.
7. **INV-7** — Scheduler walks the ready set on every event from the plan store watch (§3.12.5).
8. **INV-8** — Concurrency admission: per-user limiter THEN host broker; failure to admit re-queues the job at the head of the ready channel.
9. **INV-9** — `PauseAsync` stops admitting new jobs but allows in-flight to complete.
10. **INV-10** — `CancelAsync` flips plan status to `Canceled`; cancels all in-flight job CTs; transitions all `Pending` jobs to `Canceled`.
