# Job 006 — Redaction

**Tier:** T2
**Producer id:** `t2-redaction`
**Dependencies:** `t1-abstractions`, `t1-testkit`
**Estimated LOC:** ~700 production, ~900 tests

## Goal

Implement `AiOrchestrator.Redaction` — `IRedactor` plus the **T3-RED-*** detector pipeline (§3.7) and the **DIAG-1 / DIAG-RECIP-*** pseudonymization layer (§3.31.1.3). All regexes compiled via `[GeneratedRegex]` source generator. Detectors stateless and thread-safe. Pseudonymizer supports `Off`, `Anonymous` (one-way SHA-256-truncated), and `Reversible` (operator-key-decrypted) modes.

## Source-doc sections

- §3.7 Redaction policy
- §3.31.1.3 Diagnose pseudonymization (DIAG-1, DIAG-RECIP-*)
- §3.31.1.5 P-SID-2 anchored regex

### Behavioral invariants

1. **INV-1** — Each detector exposes a stable `RuleId` matching its source-doc reference (T3-RED-1..7, P-SID-2).
2. **INV-2** — All regexes via `[GeneratedRegex]` (zero runtime regex compilation; OE0008-style rule).
3. **INV-3** — `Redact` is idempotent: `Redact(Redact(s)) == Redact(s)`.
4. **INV-4** — `Redact` never lengthens the secret token (replacement is always shorter or equal length to detected match plus a fixed marker).
5. **INV-5** — P-SID-2 regex is anchored — does NOT match `S-1-5-21-...` substring inside a longer token like `MyS-1-5-21-thingId` (regression for v1.1 finding).
6. **INV-6** — Reversible-mode pseudonymizer encrypts the reverse table with the operator's public key (RSA-OAEP-SHA-256); the encrypted blob is the only artifact persisted to disk by the diagnose bundle.
7. **INV-7** — Anonymous-mode mapping is deterministic: same input always yields same SHA-256-truncated mapped value within a single bundle (uses a per-bundle salt).
8. **INV-8** — Async `RedactAsync` operates on `ReadOnlySequence<byte>` without materializing strings on the happy path.
9. **INV-9** — Detectors run in parallel inside `Redact` for inputs > 16 KB.
