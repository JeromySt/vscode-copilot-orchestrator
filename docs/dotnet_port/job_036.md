## Goal

`AiOrchestrator.Bindings.Node` per §3.17 — N-API bindings exposing the .NET orchestrator API to Node.js callers (VS Code extension host). Implements HOST-SCOPE-* (scoped object lifetime), shared-memory result transport for large payloads, async iteration over events.

### Public API surface

```csharp
// .NET side: AiOrchestrator.Bindings.Node
namespace AiOrchestrator.Bindings.Node;

public sealed class NodeBindingsHost : IAsyncDisposable
{
    public NodeBindingsHost(IServiceProvider sp);
    public ValueTask<HandleId> CreateHandleAsync<T>(T instance, CancellationToken ct) where T : class;
    public ValueTask<T> ResolveHandleAsync<T>(HandleId handle, CancellationToken ct) where T : class;
    public ValueTask DisposeHandleAsync(HandleId handle, CancellationToken ct);
}
public readonly record struct HandleId(long Value);

public sealed class HandleScope : IAsyncDisposable
{
    public HandleScope(NodeBindingsHost host);
    public ValueTask<HandleId> RegisterAsync<T>(T instance, CancellationToken ct) where T : class;
    public ValueTask DisposeAsync();   // disposes all handles registered in this scope
}

public sealed class SharedMemoryRingBuffer : IAsyncDisposable
{
    public SharedMemoryRingBuffer(string name, int capacity);
    public ValueTask<int> WriteAsync(ReadOnlyMemory<byte> data, CancellationToken ct);
    public ValueTask<int> ReadAsync(Memory<byte> buffer, CancellationToken ct);
    public ValueTask DisposeAsync();
}
```

### Behavioral invariants

1. **INV-1** (HOST-SCOPE-1) — Every .NET object exposed to Node lives behind a `HandleId`; Node never has a direct reference. Garbage collection on the .NET side is governed by `HandleScope` disposal.
2. **INV-2** (HOST-SCOPE-2) — `HandleScope` is `IAsyncDisposable` (and Node `AsyncDisposable`); disposal releases ALL handles registered within.
3. **INV-3** (HOST-SCOPE-3) — Handle resolution after disposal raises `HandleDisposedException` / `Error('handle disposed')`.
4. **INV-4** — Large result payloads (> 1 MiB) are shipped via `SharedMemoryRingBuffer` (Linux: `shm_open` + `mmap`; Windows: `CreateFileMapping`) to avoid Node ↔ .NET marshaling overhead.
5. **INV-5** — Async iterators (`watch()`) push events via Node's `AsyncIterator` protocol; backpressure handled by ring buffer fill.
6. **INV-6** — Node bindings are TypeScript-typed; `.d.ts` is generated from .NET public surface via a build step.
7. **INV-7** — N-API version targeted: 8 (Node 16+ LTS).
8. **INV-8** — Errors crossing the boundary are typed (`code` field maps to .NET exception type name).
