# Job 009 — File System

**Tier:** T2
**Producer id:** `t2-filesystem`
**Dependencies:** `t1-abstractions`, `t1-testkit`, `t2-path-validator`
**Estimated LOC:** ~700 production (incl. native interop), ~900 tests

## Goal

Implement `AiOrchestrator.FileSystem` — `IFileSystem` providing path-validated, async, async-disposable file I/O. Always validates inputs through `IPathValidator`. New files use `O_EXCL` + 0600 perms (CMD-TMP-1). Replacement uses atomic rename. `IFileWatcher` debounces. `GetMountKindAsync` implements **NFS-DET-*** detection.

### Behavioral invariants

1. **INV-1** — Every API canonicalizes input via `IPathValidator.AssertSafe` against the configured `AllowedRootSet`.
2. **INV-2** — `OpenWriteExclusiveAsync` opens with `FileMode.CreateNew` (Linux/macOS: O_EXCL+O_CREAT, Windows: CREATE_NEW); throws `IOException` if file exists.
3. **INV-3** — On Linux/macOS, `OpenWriteExclusiveAsync` sets file mode to 0600 (or the supplied `FilePermissions`) atomically at create time (`open(..., O_CREAT|O_EXCL, mode)`).
4. **INV-4** — On Windows, `OpenWriteExclusiveAsync` writes a DACL granting only the current user.
5. **INV-5** — `MoveAtomicAsync` uses `rename(2)` on POSIX / `MoveFileEx` with `MOVEFILE_REPLACE_EXISTING|MOVEFILE_WRITE_THROUGH` on Windows.
6. **INV-6** — `GetMountKindAsync`: on Linux returns `Nfs` for fstype `nfs/nfs4`, `Smb` for `cifs/smb3`, `Local` otherwise; on Windows, returns `Nfs` for UNC paths to NFS shares, `Smb` for UNC paths to SMB, `Local` otherwise.
7. **INV-7** — `DebouncedFileWatcher` collapses bursts within the debounce window; emits ONE event per affected path per window.
8. **INV-8** — All sync `File.*`/`Directory.*` calls inside this project's namespace are documented in code comments as deliberate (project IS the implementation behind `IFileSystem`); analyzer **OE0011** is suppressed for this project only.
