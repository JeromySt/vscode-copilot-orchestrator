# Job 015 — Line View

**Tier:** T3

**Estimated LOC:** ~500 production, ~700 tests

## Goal

`AiOrchestrator.LineView` — deterministic, replayable, non-blocking projection of process stdout/stderr into "live tail by line" semantics for UIs and agent runners. Implements **LV-***, **LV-ASYNC-***, and `LineSnapshot` (§3.4.7). Handles ANSI control sequences correctly (no split inside CSI), `\r\n` / `\n` / bare `\r` correctly.

## Detailed implementation requirements

### Public API surface

```csharp
namespace AiOrchestrator.LineView;

public sealed class LineProjector : IAsyncDisposable
{
    public LineProjector(JobId jobId, IEventStore store, IEventBus bus, IClock clock, IOptionsMonitor<LineViewOptions> opts);
    public IAsyncEnumerable<LineEmitted> ReadReplayAndLiveAsync(CancellationToken ct);   // SUB-3 seam
    public LineSnapshot CurrentSnapshot { get; }
    public ValueTask DisposeAsync();
}

public sealed record LineViewOptions { public int MaxLineBytes { get; init; } = 64 * 1024; public int SnapshotMaxLines { get; init; } = 10_000; }

public sealed record LineEmitted { public required JobId JobId { get; init; } public required long LineNumber { get; init; } public required ReadOnlyMemory<byte> Utf8Bytes { get; init; } public required DateTimeOffset At { get; init; } public required LineStream Stream { get; init; } }
public enum LineStream { StdOut, StdErr }

public sealed class LineSnapshot
{
    public long FirstLineNumber { get; }
    public long LastLineNumber { get; }
    public IReadOnlyList<LineEmitted> Lines { get; }   // bounded by SnapshotMaxLines (oldest dropped)
}

internal sealed class AnsiSafeSplitter
{
    public int FindNextLineEnd(ReadOnlySpan<byte> input, out int eolWidth);   // returns -1 if no terminator within MaxLineBytes
    public bool IsInsideAnsiCsi(ReadOnlySpan<byte> input, int position);
}
```

### Behavioral invariants

1. **INV-1** (LV-1) — A `LineEmitted` is produced when a line terminator (`\n`, `\r\n`, or bare `\r`) is observed.
2. **INV-2** (LV-2) — Splitter does NOT cut a line inside an ANSI CSI escape sequence (`\x1b[ ... <final byte>`).
3. **INV-3** (LV-3) — Bare `\r` (CR-only progress lines) emits a `LineUpdated`-style emission via `LineNumber` re-use; `LineEmitted.Stream` carries the same number.
4. **INV-4** (LV-ASYNC-1) — `ReadReplayAndLiveAsync` honors `CancellationToken` within ≤ 100 ms.
5. **INV-5** (LV-ASYNC-2) — Replay-then-live transition produces no duplicate `LineNumber` and no gap (re-uses SUB-3 invariant from job 13).
6. **INV-6** — `MaxLineBytes` cap: lines exceeding the cap are emitted as a single `LineEmitted` with a continuation marker; subsequent bytes belong to the next `LineNumber`.
7. **INV-7** — `LineSnapshot` is consistent under concurrent producers (snapshot-isolation via `ImmutableArray` swap).
8. **INV-8** — Splitter operates on `ReadOnlySequence<byte>`; no `string` allocations until consumer materializes.
