# OE0007 — Async method missing CancellationToken parameter

**Severity:** Error  
**Category:** AiOrchestrator  

## Rule description

Every `async` method must accept a `CancellationToken` as the last parameter (or an optional
parameter with `= default`). This enables cooperative cancellation and prevents resource leaks
when callers need to abort an operation.

Exemptions:
- Entry-point `Main` methods.
- `async void` event-handler methods (flagged separately by OE0008).
- Interface implementations where the interface member itself lacks a token.

## Non-compliant code

```csharp
// ❌ No CancellationToken
public async Task DoWorkAsync() // OE0007
{
    await Task.Delay(100);
}
```

## Compliant code

```csharp
// ✅ CancellationToken as last parameter
public async Task DoWorkAsync(CancellationToken cancellationToken = default)
{
    await Task.Delay(100, cancellationToken);
}
```

## Code fix

The bundled fix appends `CancellationToken cancellationToken = default` to the parameter list.
