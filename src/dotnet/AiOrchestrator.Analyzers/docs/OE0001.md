# OE0001 — Public type or member missing XML documentation comment

**Severity:** Error  
**Category:** AiOrchestrator  

## Rule description

Every public type (class, struct, interface, enum, delegate, record) and every public member
(method, property, field, event) must carry an XML documentation comment (`/// <summary>…</summary>`).
Well-documented public APIs reduce onboarding friction and make IDE tooltips useful.

## Non-compliant code

```csharp
// ❌ Missing XML doc on public class
public class MyService
{
    // ❌ Missing XML doc on public method
    public void DoWork() { }
}
```

## Compliant code

```csharp
/// <summary>Performs work on behalf of the orchestrator.</summary>
public class MyService
{
    /// <summary>Executes the work unit synchronously.</summary>
    public void DoWork() { }
}
```

## Suppression

Suppress only when the member is a trivial override or generated code that cannot carry docs.

```csharp
[SuppressMessage("AiOrchestrator", "OE0001", Justification = "Generated override.")]
public override string ToString() => base.ToString()!;
```

## Code fix

The bundled code fix inserts a `/// <summary>TODO: Add documentation.</summary>` template above
the offending declaration. Replace the TODO placeholder before committing.
