# OE0009 — [DllImport] used instead of [LibraryImport]

**Severity:** Error  
**Category:** AiOrchestrator  

## Rule description

`[DllImport]` generates P/Invoke stubs using runtime reflection-based marshalling.
`[LibraryImport]` (added in .NET 7) generates source-level marshalling code that is
trimming-safe, NativeAOT-compatible, and generally faster.

All new P/Invoke declarations must use `[LibraryImport]`.

## Non-compliant code

```csharp
// ❌ Legacy DllImport
[DllImport("user32.dll")] // OE0009
public static extern bool ShowWindow(IntPtr hWnd, int nCmdShow);
```

## Compliant code

```csharp
// ✅ Source-generated LibraryImport
[LibraryImport("user32.dll")]
[return: MarshalAs(UnmanagedType.Bool)]
public static partial bool ShowWindow(IntPtr hWnd, int nCmdShow);
```

Note: the enclosing class must be `partial`.

## Code fix

The bundled fix replaces `[DllImport(...)]` with `[LibraryImport(...)]` and adds the `partial`
keyword to the method and its enclosing class if not already present.
