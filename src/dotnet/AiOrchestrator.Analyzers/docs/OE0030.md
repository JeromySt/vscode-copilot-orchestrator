# OE0030 — Test method missing [ContractTest] attribute

**Severity:** Warning  
**Category:** AiOrchestrator  

## Rule description

Every test method decorated with `[Fact]` or `[Theory]` in a `tests/dotnet/**` project must also
carry `[ContractTest("RULE-ID")]`. This links the test to a specific acceptance criterion and
enables the `check-contract-tests.ps1` script to verify coverage.

## Non-compliant code

```csharp
// ❌ [Fact] without [ContractTest]
[Fact]
public void SomeTest() { } // OE0030
```

## Compliant code

```csharp
// ✅ Both attributes present
[Fact]
[ContractTest("OE0001")]
public void OE0001_PositiveCase_FlagsMissingDoc() { }
```

## Code fix

The bundled fix inserts `[ContractTest("TODO")]` above the test method. Replace the `"TODO"`
placeholder with the actual rule or criterion ID before committing.

## Suppression

Suppress only for helper methods that are inadvertently decorated with `[Fact]`.
