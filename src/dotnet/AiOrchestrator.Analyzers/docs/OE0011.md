# OE0011 — Synchronous File I/O API

**Severity:** Error  
**Category:** AiOrchestrator  

## Rule description

Synchronous `System.IO.File` read/write methods block the calling thread for the duration of the
I/O operation. Under high concurrency this starves the thread pool. Use async alternatives or the
`IFileSystem` abstraction.

Banned synchronous methods:
- `File.ReadAllText` → `File.ReadAllTextAsync` or `IFileSystem.ReadAllTextAsync`
- `File.WriteAllText` → `File.WriteAllTextAsync` or `IFileSystem.WriteAllTextAsync`
- `File.Open` / `File.OpenRead` / `File.OpenWrite` → async stream APIs

## Non-compliant code

```csharp
// ❌ Blocking thread on I/O
public string Load(string path)
    => File.ReadAllText(path); // OE0011
```

## Compliant code

```csharp
// ✅ Non-blocking async I/O
public Task<string> LoadAsync(string path, CancellationToken ct)
    => _fs.ReadAllTextAsync(path, ct);
```
