# OE0012 — Process.Start is banned

**Severity:** Error  
**Category:** AiOrchestrator  

## Rule description

`Process.Start` directly spawns an OS process, bypassing the `IProcessSpawner` abstraction. This
makes code untestable and prevents the orchestrator from monitoring child-process lifecycle (CPU,
memory, cancellation, exit code capture).

## Non-compliant code

```csharp
// ❌ Direct process spawn
using System.Diagnostics;
public class Launcher
{
    public void Run(string exe)
        => Process.Start(exe); // OE0012
}
```

## Compliant code

```csharp
// ✅ Use IProcessSpawner from DI
public class Launcher
{
    private readonly IProcessSpawner _spawner;
    public Launcher(IProcessSpawner spawner) => _spawner = spawner;
    public Task<int> RunAsync(string exe, CancellationToken ct)
        => _spawner.SpawnAsync(exe, Array.Empty<string>(), ct);
}
```
